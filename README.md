# 🧬 Bio-Graph Agent: 基于腾讯混元 Hy3 的跨文献增量生物知识图谱引擎

本项目针对 [Tencent-Hunyuan/Hy3/issues/4](https://github.com/Tencent-Hunyuan/Hy3/issues/4) 的长文本结构化抽取与知识落地诉求开发。利用 **腾讯混元 Hy3** 强大的长文本上下文理解能力与严谨的推理性能，构建了一套能够自文献（PDF）中自动提取、双层反思质检、跨文献语义对齐，并最终生成高交互性动态网络拓扑图的 **Bio-Knowledge Graph LLM Agent**。

---

## 🎯 项目立项初衷与科学价值 (Why this project?)

在生物医学科研领域，一个经典的痛点是：**如何在浩如烟海的文献中寻找极具科研价值的全新选题？**

已知在生物学中，“分子 A 调控 蛋白质 B”与“蛋白质 B 调控 生物表型/生理系统 C”可能各自散落在不同年份、不同期刊的孤立论文中。而**证明“A 是通过 B 这一关键节点来介导并影响 C”的调控级联通路，往往就是一个极具学术价值、能够直接支撑一项重大课题或基金申报的科研选题。**

然而，依靠人工阅读去拼凑这种跨文献的级联关系，效率极低且极易遗漏。

**Bio-Graph Agent** 正是为了攻克这一痛点而生。它能够作为科研人员的“智能选题军师”：
1. **输入多篇文献**：自动高效解析长篇复杂学术论文。
2. **打通知识孤岛**：通过 AI 语义对齐，将多篇不同论文里的分子、蛋白质、生理系统等实体以及它们之间的潜在关系串联并网。
3. **直观拓扑渲染**：根据具体的生物学调控逻辑，自动渲染出极具科研规范的动态拓扑关系网，帮助科研人员一眼洞察出潜在的 `A ➔ B ➔ C` 的核心调控路径。

---

## 🎨 拓扑网络可视化视觉规范

本系统严格遵循生物学调控网络图谱的专业规范进行视觉渲染，帮助科研人员快速捕捉信号通路的本质：
* **🟢 绿色连线 **：代表 **正作用 / 激活 (Activation / Promotion)**。
* **🔴 红色连线 **：代表 **负作用 / 抑制 (Inhibition / Suppression)**，在拓扑交互中一目了然。
* **⚪ 灰色虚线 (---)**：代表 **一般关联 / 有相互作用但方向未明 (Correlation)**。
* **⚪ 紫色连线 (---)**：代表 **从属包含关系**
* 另有可选显示的**捷径线**
* **🔮 节点动态热度**：节点大小随着它在跨文献网络中的“连接度（热度）”自动放大。鼠标悬停即可查看其所有跨文献别名、支持该关系的原文证据（Evidence）以及精准的文献来源溯源。

## 🌟 核心功能与操作指南
### ⚙️ 一、 初始环境与检索设置
🌐 灵活的大模型引擎：支持腾讯云、OpenRouter、硅基流动等云端 API，同时原生支持 Ollama 本地断网运行，保护极密医学数据。

📚 智能检索源：内置 PubMed (医学绝对权威) 与 Semantic Scholar (全科 AI 智能极速检索) 双引擎。

### 📂 二、 知识图谱构建 (双模启动)
💡 零基础冷启动：无需本地文献，只需用自然语言输入研究方向（如“巨噬细胞极化在肿瘤微环境中的作用”），AI 将自动生成检索式、抓取外网文献并直接搭建第一张图谱。

📄 本地文献精准解析：拖入单篇 PDF，支持指定页码解析、摘要极速模式。开启“🔗 追加模式”后，新解析的文献不会覆盖现有图谱，而是与之跨界融合、无限生长。

### 🎨 三、 视觉赋权引擎与专业拓扑规范
图谱生成后，系统利用四大硬核视觉算法，在一团乱麻中找出科研靶点：

🟢 绿色实线：正作用 / 促进 / 激活 (Activation)。

🔴 红色实线：负作用 / 抑制 / 下调 (Inhibition)。

🟣 紫色实线：包含 / 归属 / 组成关系 (Containment)。

⚪ 灰色实线：一般关联 / 相关性 (Correlation)。

⚠️ 灰色虚线 (捷径边)：AI 检测到跨级跳跃关系时（已知 A->B->C，又发现 A->C），会自动将粗放的 A->C 降级为灰色虚线捷径，凸显主干逻辑。

🔮 节点辐射度与主干加粗：基于图论连接度算法，核心枢纽基因的节点会自动放大；被多篇文献反复印证的共识连线会自动加粗。

### 🤖 四、 AI 智能科研助理
在右侧的 🤖 问 AI 面板，集成了四大自动化武器：

🧹 智能洗树 (Pruning)：自动发现并建议合并同义词、建立宏观/微观包含层级；并支持意图节点清洗（输入研究方向，一键抹杀偏题器官或无关分子）。

🔍 智能拓展 (Smart Expansion)：选中图谱边缘的孤岛节点，AI 自动生成 PubMed 检索式抓取前沿文献，开启“无人值守模式”将其自动对齐熔接到当前图谱。

🔗 智能桥接 (Bridging)：选中任意两个毫不相干的节点，一键呼叫算法找出它们之间隐秘的传导路径，为撰写基金标书提供终极灵感。

📖 连线深度机制解读：点击任意节点或连线，AI 结合原文上下文生成深度的生物学推演报告。

### ✂️ 五、 专家手动干预 (数据管理中心)
拥有图谱的“上帝权限”：

一键清除：清理游离的孤岛节点，或批量替换文献来源标签。

节点手术：重命名、强制合并（重命名为已有节点即可触发连线完美转移）、乃至将笼统节点硬拆分为两个并逐一分配连线。

连线修剪：逆转上下游方向、修改关联属性，或手动建立隐秘关联桥梁。

### 💾 六、 工程记忆与本地化管理 (.biokg)
安全锁机制：清空画布前带有严格的防误触二次确认。

一键导出/读档：支持将整张带有历史证据、修改记录的庞大网络导出为专属的 .biokg 存档文件，并支持随时无缝拖拽载入，方便科研成果转移与长期积累。

## 🤖 腾讯混元 Hy3 在系统中的核心角色 (Role of Hy3)

本系统并非简单地将大模型作为一个“文本总结器”，而是**深度榨取了腾讯混元 Hy3 开源模型的性能红利**。结合 Hy3 卓越的超长上下文处理能力、严谨的逻辑推理准度以及原生的 Agent 工具调用生态，我们在图谱构建的**解析前 - 融合中 - 成图后**全生命周期中，打造了六大核心智能体：
```text
┌──────────────────────────────────────────────────────────────────────┐
│                        🧠 大模型智能核心架构                          │
└──────────────────────────────────┬───────────────────────────────────┘
                                   │
          ┌────────────────────────┼────────────────────────┐
          ▼                        ▼                        ▼
【一期：极速解析与建网】      【二期：跨域对齐与洗树】      【三期：破圈生长与洞察】
  1. 长文本理解中枢            3. 跨文献对齐裁判            5. 联网拓展与桥接向导
  2. 双层反思质检官            4. 拓扑智能清洗专家          6. 随身机制解说员
```


### 1. 长文本理解中枢 (Long-Context Understanding Hub)
* **面临挑战**：生物医药领域的学术论文通常长达十几页甚至几十页，包含海量的专业术语、图表叙述和复杂的句式结构，对模型的长文本吞吐量和上下文召回率（Needle in a Haystack）要求极高。
* **混元角色**：系统利用混元 Hy3 极大的上下文窗口，让大模型充当“科学速读专家”，在不丢失前后文关联的前提下，高效率、无损地从整篇 PDF 论文中抽取标准的生物实体（基因/蛋白质/表型）和精细的调控机制。

### 2. 双层反思质检官 (Double-Layer Reflection Inspector)
* **面临挑战**：传统的单次大模型抽取容易产生学术幻觉（Hallucination），误报或捏造不存在的生物学关系，这在严谨的科学研究中是致命的。
* **混元角色**：系统构建了基于混元 Hy3 的 **“初审抽取 - 反思对质”** 双阶段 Agent。Hy3 在这里分饰两角：
  - **角色 A (提取员)**：根据正文提取初步的实体和连线。
  - **角色 B (审计员)**：利用混元极强的严谨逻辑，拿着初步结果重新审视原文，执行“原话对质”。只有在原文中找到铁证（Evidence）的关系才能保留，利用混元自身的反思能力将幻觉率降到了极低。

### 3. 跨文献语义对齐裁判 (Cross-Literature Alignment Judge)
* **面临挑战**：多篇文献独立解析后，同义词（如 `p53` / `TP53` / `Tumor Protein 53`）会导致图谱碎片化，形成大量的知识孤岛。
* **混元角色**：在追加合并模式下，混元 Hy3 承担 **“全网知识融合裁判”** 的重任。它能够突破字面意思的局限，深度理解生物学概念的语义层。通过交叉比对不同文献抽出的本体名称和别名（Aliases）列表，混元能够精准执行语义级吸合，将碎片化的跨文献连线完美织入同一张信号调控大网中。

### 4. 拓扑智能清洗专家 (Topology Cleaner)
* **面临挑战**：成图后的网络往往存在复杂的拓扑冗余，例如宏观/微观包含关系混乱、以及跨越多个层级的宽泛结论（捷径边），人工梳理这种蛛网般的拓扑结构极具挑战。且文献中常提取出研究者不关心的偏题分子或器官。

* **混元角色**：依靠混元 Hy3 强大的宏观结构分析与逻辑推导能力，它能通读全图拓扑逻辑，自动诊断并建议“降级机制捷径边”、建立层级包含关系。同时支持基于自然语言的“意图剪枝”，精准理解用户的研究方向，一键揪出并抹杀所有偏题的边缘分支。

### 5. 联网拓展与桥接向导 (Web-Browsing Explorer)
* **面临挑战**：本地文献库的知识始终存在边界。当图谱中出现孤岛节点，或者需要探寻两个看似无关分子之间的潜在传导路径时，传统方法需要人工耗费大量时间去外部数据库重新检索和阅读。

* **混元角色**：全面激活了混元 Hy3 的 原生 Tool Calling (工具调用) 与规划能力。选中图谱孤岛，Hy3 能自主生成高级检索式，呼叫外网 API (如 PubMed / Semantic Scholar) 抓取前沿文献。在“无人值守模式”下，它能自动解析新文献并将知识无缝熔接回当前图谱；对于任意两个不相干节点，更能主动寻找隐秘的传导桥梁，实现图谱的破圈生长。

### 6. 随身微观机制解说员 (Micro-Mechanism Explainer)
* **面临挑战**：高度抽象的图谱连线（如 A ➔ B）往往掩盖了原文中复杂细腻的生化反应过程。科研人员在审阅宏观大图时，难以瞬间回忆起某条具体连线背后的详细机制与实验出处。

* **混元角色**：凭借混元 Hy3 强大的中文长文本生成、润色与上下文理解能力，系统允许用户点击图谱中的任意一条连线。Hy3 会提取该连线专属的微上下文（Micro-context）和原文证据，当场生成一篇逻辑严密、极其详实的生物机制推演报告，直接为撰写科研基金标书提供文本素材。

## 🌟 核心技术亮点

1. **长文级实体与关系结构化抽取**
   - 完美适配混元 Hy3 的超长上下文窗口，能直接解析几十页的复杂生物化学/医学 PDF 文献。
   - 提取包含 `标准名称`、`别名列表(Aliases)`、`关系类型` 及 `原文证据(Evidence)` 的高质量结构化 JSON 数据。

2. **首创 Agent 双层反思质检机制 (Reflection Mechanism)**
   - **初审抽取**：第一层 Agent 负责粗粒度的大网捕获。
   - **反思对质**：第二层 Agent 作为“独立质检员”，将初审结果与原文一一对照，无证据的幻觉实体一律剔除，极大提升了图谱的严谨性。

3. **跨文献智能实体对齐融合中枢 (Entity Aligner)**
   - 开启**追加模式**后，多篇文献的知识脉络可动态并网。
   - 呼叫混元大模型作为裁判，并**交叉参考实体自带的别名（Aliases）列表**，实现高精度的语义对齐。例如：自动将文献 A 中的 `p53` 与文献 B 中的 `Tumor protein 53` / `TP53` 识别并吸合为同一个全局超级节点，并完美拼接各自的文献来源标签（如：`文献A.pdf | 文献B.pdf`）。

4. **无外部网络依赖的动态拓扑渲染**
   - 基于 Pyvis 构建，针对 Streamlit 嵌入式网页渲染进行了极致优化。
   - 节点大小随“热度（连接度中心性）”动态缩放，鼠标悬停可查看其所有跨文献别名、关系证据及文献出处。

5. **基于大模型推导的智能洗树与意图剪枝 (Intelligent Pruning & Intent Cleansing)**
   - **拓扑智能洗树**：AI 自动通读全图逻辑，识别并建议合并遗漏的同义词、建立宏观包含层级，并将跨层级的“宽泛结论”智能降级为灰色虚线（机制捷径边），凸显核心主干。
   - **自然语言意图剪枝**：支持用大白话输入研究焦点（如“我只关注肺癌机制”），大模型将化身严苛的审稿人，自动揪出图谱中偏题的器官或旁支分子，实现一键物理抹杀，保障图谱极度纯净。

6. **突破本地边界的自动化生长与桥接 (Automated Expansion & Bridging)**
   - **无人值守拓展**：选中图谱孤岛节点，系统自动结合图谱上下文生成专业检索式，呼叫外网 API (如 PubMed) 抓取前沿文献进入队列，并自动完成新知识与当前图谱的完美熔接。
   - **跨节点智能桥接**：指定任意两个毫不相干的节点，系统依靠图论算法与大模型外网挖掘，强行探勘两者之间的多步级联传导路径，直接启发全新的科研选题。

7. **细粒度微上下文机制解说 (Micro-Context Mechanism Explanation)**
   - 突破传统图谱“只能看线，不知道为什么连线”的痛点。
   - 深度交互：点击图谱中的任意节点或连线，系统会实时提取该部位的提取依据与原始文献来源（微上下文），呼叫大模型当场推演并生成极其详实的生物学机制解读报告，直出标书级素材。

8. **工业级状态机管理与双语工程底座 (State Machine & i18n Foundation)**
   - **专属科研资产沉淀**：支持将携带海量证据链的动态大网及人类专家手动修改的记录，一键导出为私有化的 `.biokg` 存档，支持随时断点读档、无缝渲染。
   - **动态国际化架构**：底层采用标准的 i18n 动态双语注入框架，结合严密的 Streamlit 状态机流转逻辑，实现全界面与大模型输出结果的中英双语无缝热切换。

## 🚀 快速开始
1. 克隆仓库与安装依赖

```text
git clone https://github.com/24238-T-34/-Bio-Graph-Agent-Hy3-
cd -Bio-Graph-Agent-Hy3-
pip install -r requirements.txt
```

---

2. 配置并运行项目

方法一（Linux/WSL）

装载仓库后运行
```text
chmod +x run.sh
./run.sh
```

方法二（Windows)

点击批处理文件 start.bat

*项目路径可能有误，必要时劳请手动修改

方法三

运行

```text
streamlit run app.py
```
