# 🧬 Bio-Graph Agent: 基于腾讯混元 Hy3 的跨文献增量生物知识图谱引擎

本项目针对 [Tencent-Hunyuan/Hy3/issues/4](https://github.com/Tencent-Hunyuan/Hy3/issues/4) 的长文本结构化抽取与知识落地诉求开发。利用 **腾讯混元 Hy3** 强大的长文本上下文理解能力与严谨的推理性能，构建了一套能够自文献（PDF）中自动提取、双层反思质检、跨文献语义对齐，并最终生成高交互性动态网络拓扑图的 **Bio-Knowledge Graph LLM Agent**。

---

## 🎯 项目立项初衷与科学价值 (Why this project?)

在生物医学科研领域，一个经典的痛点是：**如何在浩如烟海的文献中寻找极具科研价值的全新选题？**

已知在生物学中，“分子 A 调控 蛋白质 B”与“蛋白质 B 调控 生物表型/生理系统 C”可能各自散落在不同年份、不同期刊的孤立论文中。而**证明“A 是通过 B 这一关键节点来介导并影响 C”的调控级联通路，往往就是一个极具学术价值、能够直接支撑一项重大课题或基金申报的科研选题。**

然而，依靠人工阅读去拼凑这种跨文献的级联关系，效率极低且极易遗漏。

**Bio-Graph Agent** 正是为了攻克这一痛点而生。它能够作为科研人员的“智能选题军师”：
1. **输入多篇文献**：自动高效解析长篇复杂学术论文。
2. **打通知识孤岛**：通过 AI 语义对齐，将多篇不同论文里的分子、蛋白质、生理系统等实体以及它们之间的潜在关系串联并网。
3. **直观拓扑渲染**：根据具体的生物学调控逻辑，自动渲染出极具科研规范的动态拓扑关系网，帮助科研人员一眼洞察出潜在的 `A ➔ B ➔ C` 的核心调控路径。

---

## 🎨 拓扑网络可视化视觉规范

本系统严格遵循生物学调控网络图谱的专业规范进行视觉渲染，帮助科研人员快速捕捉信号通路的本质：
* **🟢 绿色连线 **：代表 **正作用 / 激活 (Activation / Promotion)**。
* **🔴 红色连线 **：代表 **负作用 / 抑制 (Inhibition / Suppression)**，在拓扑交互中一目了然。
* **⚪ 灰色虚线 (---)**：代表 **一般关联 / 有相互作用但方向未明 (Correlation)**。
* **🔮 节点动态热度**：节点大小随着它在跨文献网络中的“连接度（热度）”自动放大。鼠标悬停即可查看其所有跨文献别名、支持该关系的原文证据（Evidence）以及精准的文献来源溯源。

## 🤖 腾讯混元 Hy3 在系统中的核心角色 (Role of Hy3)

本系统并非简单地调用大模型，而是深度结合了 **腾讯混元 Hy3 (Tencent Hunyuan Hy3)** 的超长上下文处理能力与强悍的逻辑推理性能，将其作为系统的**三大核心智能体中枢**：
```text
┌┌────────────────────────────────────────────────────────┐
│               🧠 腾讯混元 Hy3 智能核心                  │
└───────────────────────────┬────────────────────────────┘
│
┌─────────────────────┼─────────────────────┐
▼                     ▼                     ▼
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│ 长文本理解中枢 │     │ 双层反思质检官 │     │ 跨文献对齐裁判 │
└───────────────┘     └───────────────┘     └───────────────┘
```


### 1. 长文本理解中枢 (Long-Context Understanding Hub)
* **面临挑战**：生物医药领域的学术论文通常长达十几页甚至几十页，包含海量的专业术语、图表叙述和复杂的句式结构，对模型的长文本吞吐量和上下文召回率（Needle in a Haystack）要求极高。
* **混元角色**：系统利用混元 Hy3 极大的上下文窗口，让大模型充当“科学速读专家”，在不丢失前后文关联的前提下，高效率、无损地从整篇 PDF 论文中抽取标准的生物实体（基因/蛋白质/表型）和精细的调控机制。

### 2. 双层反思质检官 (Double-Layer Reflection Inspector)
* **面临挑战**：传统的单次大模型抽取容易产生学术幻觉（Hallucination），误报或捏造不存在的生物学关系，这在严谨的科学研究中是致命的。
* **混元角色**：系统构建了基于混元 Hy3 的 **“初审抽取 - 反思对质”** 双阶段 Agent。Hy3 在这里分饰两角：
  - **角色 A (提取员)**：根据正文提取初步的实体和连线。
  - **角色 B (审计员)**：利用混元极强的严谨逻辑，拿着初步结果重新审视原文，执行“原话对质”。只有在原文中找到铁证（Evidence）的关系才能保留，利用混元自身的反思能力将幻觉率降到了极低。

### 3. 跨文献语义对齐裁判 (Cross-Literature Alignment Judge)
* **面临挑战**：多篇文献独立解析后，同义词（如 `p53` / `TP53` / `Tumor Protein 53`）会导致图谱碎片化，形成大量的知识孤岛。
* **混元角色**：在追加合并模式下，混元 Hy3 承担 **“全网知识融合裁判”** 的重任。它能够突破字面意思的局限，深度理解生物学概念的语义层。通过交叉比对不同文献抽出的本体名称和别名（Aliases）列表，混元能够精准执行语义级吸合，将碎片化的跨文献连线完美织入同一张信号调控大网中。


## 🌟 核心技术亮点

1. **长文级实体与关系结构化抽取**
   - 完美适配混元 Hy3 的超长上下文窗口，能直接解析几十页的复杂生物化学/医学 PDF 文献。
   - 提取包含 `标准名称`、`别名列表(Aliases)`、`关系类型` 及 `原文证据(Evidence)` 的高质量结构化 JSON 数据。

2. **首创 Agent 双层反思质检机制 (Reflection Mechanism)**
   - **初审抽取**：第一层 Agent 负责粗粒度的大网捕获。
   - **反思对质**：第二层 Agent 作为“独立质检员”，将初审结果与原文一一对照，无证据的幻觉实体一律剔除，极大提升了图谱的严谨性。

3. **跨文献智能实体对齐融合中枢 (Entity Aligner)**
   - 开启**追加模式**后，多篇文献的知识脉络可动态并网。
   - 呼叫混元大模型作为裁判，并**交叉参考实体自带的别名（Aliases）列表**，实现高精度的语义对齐。例如：自动将文献 A 中的 `p53` 与文献 B 中的 `Tumor protein 53` / `TP53` 识别并吸合为同一个全局超级节点，并完美拼接各自的文献来源标签（如：`文献A.pdf | 文献B.pdf`）。

4. **无外部网络依赖的动态拓扑渲染**
   - 基于 Pyvis 构建，针对 Streamlit 嵌入式网页渲染进行了极致优化。
   - 节点大小随“热度（连接度中心性）”动态缩放，鼠标悬停可查看其所有跨文献别名、关系证据及文献出处。

## 📂 核心代码架构

```text
├── app.py           # Streamlit 前端交互大盘（包含跨文献实体融合手术逻辑）
├── back_logic.py     # 系统总调度管道（BioGraphPipeline 增量累加中枢）
├── LLM_SYS.py       # 大模型智能体（包含初审、反思、跨文献别名增强对齐 Agent）
├── IO_SYS.py        # 基础设施层（包含 PyMuPDF 智能切块与安全图谱拓扑渲染器）
├── WebSearcher.py   #网上文献搜索功能
├── requirements.txt # 项目依赖
└── .gitignore       # Git 提交忽略配置
```

## 🚀 快速开始
1. 克隆仓库与安装依赖

```text
git clone https://github.com/24238-T-34/-Bio-Graph-Agent-Hy3-
cd -Bio-Graph-Agent-Hy3-
pip install -r requirements.txt
```

---

2. 配置并运行项目

方法一（Linux/WSL）

装载仓库后运行
```text
chmod +x run.sh
./run.sh
```

方法二（Windows)

点击批处理文件 start.bat

*项目路径可能有误，必要时劳请手动修改

方法三

运行

```text
streamlit run app.py
```

---

运行后在浏览器中打开 Streamlit 界面，在侧边栏填入您的 OpenRouter / 腾讯混元 API Key，即可开始上传 PDF 文献，见证动态生物大图谱的自动构建！
